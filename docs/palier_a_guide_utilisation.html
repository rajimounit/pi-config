1
Session courte, question ciblée — lancer Pi sans changer de modèle. qwen3.6:35b-a3b par défaut, autoCompaction gère le contexte.
2
Coding AVV Assist Pro — lancer Pi dans le repo. Ctrl+P → qwen3-coder:latest. lsp-pi corrige les erreurs TS en temps réel.
3
Analyse d'architecture / debug profond — Ctrl+P → deepseek-r1:32b. Raisonnement structuré sur des problèmes non-triviaux.
4
Lecture rapide / grep / lookup — Ctrl+P → qwen2.5-coder:7b. Réponse en secondes, faible VRAM.
5
Session longue sur gros repo — démarrer llama-server (Palier B), puis Ctrl+P → llama-cpp/qwen3-coder. 64k tokens, aucune troncature.
Idées à exploiter maintenant
Enrichir AGENTS.md avec des règles spécifiques à AVV Assist Pro — ex: "ne jamais exposer de clé API en frontend", "toujours valider le JSON LLM avant injection base". Pi les respectera sur tous tes projets.
Créer un AGENTS.md par projet dans le repo AVV Assist Pro — règles locales qui s'ajoutent au global. Pi lit les deux en cascade.
Profiter du lsp-pi sur NestJS — demander à Pi de refactorer un module entier de backend. Le LSP va valider TypeScript en continu et réduire les cycles d'aller-retour à zéro.
Tester le switching de modèle mid-session — commencer en qwen3-coder pour écrire du code, Ctrl+P → deepseek-r1 pour valider l'architecture, retour qwen3-coder pour implémenter. Le contexte est conservé entre les switches.
Ajouter un modèle vision aux favorites — inclure ollama/qwen3-vl:8b dans la liste favorites pour analyser des screenshots ou maquettes directement dans Pi.
