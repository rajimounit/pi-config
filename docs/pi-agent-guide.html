Workflows opérationnels
01
Session standard — Coding AVV Assist Pro
Usage quotidien
1
Ouvrir un terminal WSL
Ollama tourne en arrière-plan — rien à démarrer.
2
Se placer dans le repo AVV Assist Pro
cd ~/git/local-setup/avvAssistv3/avv-assist-pro-v3 pi
Pi démarre avec pi-qwen3-35b par défaut + AGENTS.md chargé automatiquement.
3
Optionnel : basculer vers pi-qwen3-coder pour du coding ciblé
Ctrl+P → ollama/pi-qwen3-coder. Idéal pour écrire/modifier un module NestJS avec lsp-pi actif.
4
Travailler normalement
lsp-pi retourne les erreurs TypeScript après chaque édition. autoCompaction gère le contexte. Aucune intervention manuelle.
02
Session longue — Lecture gros repo / 64k context
llama.cpp requis
Fermer LM Studio avant de continuer — il occupe ~1.5GB de VRAM partagée.
1
Démarrer llama-server (terminal dédié)
~/.pi/start-llama.sh # Attendre : "model loaded" + "listening on :8081" # Chargement ~2 minutes (Qwen3-Coder-30B-A3B)
2
Vérifier que le serveur répond
curl -s http://127.0.0.1:8081/health # Attendu : {"status":"ok"}
3
Lancer Pi dans le repo cible
cd ~/git/local-setup/avvAssistv3/avv-assist-pro-v3 pi
4
Basculer sur llama-cpp
Ctrl+P → llama-cpp/qwen3-coder:latest. Vérifier "(llama-cpp) qwen3-coder:latest" en bas à droite.
5
Travailler avec 65 000 tokens de contexte
Pi peut lire l'ensemble du backend NestJS en une passe. Aucune troncature silencieuse.
03
Debug d'architecture — deepseek-r1
Raisonnement
1
Lancer Pi normalement
Depuis le repo concerné.
2
Switcher vers deepseek-r1:32b
Ctrl+P → ollama/deepseek-r1:32b. Idéal pour : décision d'architecture, analyse d'un bug non-trivial, validation d'approche.
3
Retourner sur qwen3-coder pour l'implémentation
Ctrl+P → ollama/pi-qwen3-coder. Le contexte de l'analyse est conservé, Pi implémente avec les conclusions de deepseek.
04
Mise à jour de Pi Agent
Maintenance
Pi v0.74.1 est disponible. Tester dans une session isolée avant d'adopter définitivement.
1
Vérifier le changelog avant de mettre à jour
pi changelog # Ou : https://github.com/earendil-works/pi-mono/blob/main/packages/coding-agent/CHANGELOG.md
2
Sauvegarder la config Pi
cd ~/.pi && git add -A && git commit -m "chore: snapshot avant update pi"
3
Mettre à jour
pi update
4
Vérifier que les packages sont toujours actifs
pi list # Tous les packages doivent apparaître
5
Committer l'état post-update
cd ~/.pi && git add -A && git commit -m "chore: pi update vX.XX.X" git push
